# /review

## Description

현재 변경된 파일(또는 지정한 파일)에 대해 코드 리뷰를 수행합니다.
프론트엔드 프로젝트 컨벤션 준수, 타입 안전성, 가독성 관점에서 검토합니다.

## Steps

1. 검토 대상 결정:
   - 인자가 있으면 해당 파일 검토 (예: `/review src/components/UserCard.tsx`)
   - 인자 없으면 `git diff dev...HEAD --name-only`로 변경 파일 목록 확인
2. 각 파일을 읽고 아래 체크리스트 기준으로 검토
3. 문제 발견 시 파일명:라인번호 형태로 위치 명시
4. 심각도 표시: 🔴 (필수 수정) / 🟡 (권장) / 🟢 (칭찬/참고)

## 리뷰 체크리스트

### TypeScript

- [ ] `any` 사용 여부 → `unknown` + 타입 가드로 교체 권고
- [ ] `import type` 미사용 여부
- [ ] 객체 타입에 `interface` 사용 여부

### 네이밍

- [ ] 변수/함수 camelCase 준수
- [ ] 컴포넌트/타입 PascalCase 준수
- [ ] 상수 UPPER_SNAKE_CASE 준수
- [ ] 커스텀 훅 `use` 접두사 준수

### 스타일

- [ ] 인라인 스타일 사용 여부
- [ ] SCSS 모듈 사용 여부
- [ ] 최상위 클래스명 = 컴포넌트명 준수

### 상태 관리

- [ ] 전역 상태에 Zustand 사용 여부
- [ ] 서버 상태에 TanStack Query 사용 여부

### 코드 품질

- [ ] 불필요한 주석 여부
- [ ] 단일 책임 원칙 위반 여부 (컴포넌트/함수가 너무 많은 역할)
- [ ] 미래를 위한 과도한 추상화 여부
- [ ] 중복 코드 (기존 컴포넌트/훅 재사용 가능 여부)

## Output 형식

```
## 코드 리뷰: <파일명>

### 🔴 필수 수정
- [파일:라인] 문제 설명 + 수정 방향

### 🟡 권장 개선
- [파일:라인] 개선 제안

### 🟢 잘된 점
- 구체적으로 언급

---
**총평**: 한 줄 요약
```
