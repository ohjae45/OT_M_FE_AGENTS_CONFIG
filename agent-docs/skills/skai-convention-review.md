# /skai-convention-review

## 설명

현재 변경된 파일(또는 지정한 파일)에 대해 컨벤션 중심 코드 리뷰를 수행합니다.
프론트엔드 프로젝트 컨벤션 준수, 타입 안전성, 가독성 관점에서 검토합니다.
사용자가 "컨벤션 리뷰", "스타일 리뷰", "타입/가독성 점검", "파일 컨벤션 확인"처럼 요청할 때 사용합니다.

## 중단 조건

- 사용자가 검토할 파일을 지정했는데 해당 파일이 존재하지 않으면 즉시 중단하고 사용자에게 알린다.
- 사용자가 검토할 파일을 지정하지 않았고 `dev` 대비 변경 파일도 없으면 즉시 중단하고 사용자에게 알린다.

## 참고 규칙 문서

리뷰 전에 프로젝트 루트 기준으로 아래 규칙 문서를 먼저 읽는다.
문서가 없으면 해당 영역은 추측해서 지적하지 않고 "확인 불가"로 분리한다.

| 영역            | 문서                                          |
| --------------- | --------------------------------------------- |
| TypeScript      | `agent-docs/rules/typescript-rules.md`       |
| 프론트엔드 공통 | `agent-docs/rules/frontend-common-rules.md`  |
| 스타일링        | `agent-docs/rules/styling-rules.md`          |
| 상태 관리       | `agent-docs/rules/state-management-rules.md` |
| 워크플로우      | `agent-docs/rules/workflow-rules.md`         |

## 진행 단계

1. 검토 대상 결정:
   - 사용자가 파일을 지정했으면 해당 파일 검토 (예: `/skai-convention-review src/components/UserCard.tsx`)
   - 사용자가 파일을 지정하지 않았으면 `git diff dev...HEAD --name-only`로 변경 파일 목록 확인
2. 참고 규칙 문서를 먼저 읽고, 확인된 규칙만 리뷰 기준으로 사용
3. 각 파일을 읽고 참고 규칙 문서 기준으로 검토
4. 문제 발견 시 파일명:라인번호 형태로 위치 명시
5. 심각도 표시: 🔴 (필수 수정) / 🟡 (권장) / 🟢 (칭찬/참고)

## 작업 원칙

- 문제 발견 시 파일명과 라인번호를 함께 명시한다.
- 실제 변경 diff와 관련된 위험을 우선 검토한다.
- 스타일 취향보다 버그, 타입 안정성, 유지보수성 문제를 우선한다.
- 확신이 낮은 내용은 단정하지 않고 확인 필요 사항으로 분리한다.
- 이 스킬은 리뷰 전용이며, 사용자가 명시적으로 요청하지 않으면 코드를 수정하지 않는다.

## 출력 형식

```
## 컨벤션 리뷰: <파일명>

### 확인 불가
- [영역] 문서 없음 또는 기준 확인 불가

### 🔴 필수 수정
- [파일:라인] 문제 설명 + 수정 방향

### 🟡 권장 개선
- [파일:라인] 개선 제안

### 🟢 잘된 점
- 구체적으로 언급

---
**총평**: 한 줄 요약
```
