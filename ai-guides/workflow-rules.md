# 작업 흐름 규칙

이 문서는 작업 전후 체크 기준과 검증 규칙을 정의한다.

## 작업 전 체크리스트

- 기존 컴포넌트, 훅, 유틸 재사용 가능 여부를 먼저 확인한다.
- 기존 스타일 재사용을 검토할 때는 먼저 같은 역할의 공용 UI인지 확인한다.
- 재사용 대상이 특정 컴포넌트 전용 스타일 파일이면 그대로 import하지 말고, 공용 스타일로 추출할지 먼저 판단한다.
- 새 패키지 추가가 필요하면 반드시 `pnpm`을 사용한다.
- 타입 안전성을 확보한다.
- 스타일은 SCSS 모듈로 작성한다.

## 검증 규칙

- 작업 후 아래 명령어를 우선 확인한다.
  - `pnpm lint`
  - `pnpm typecheck`
- 필요 시 `pnpm build`까지 확인한다.

## Pre-push Hook

푸시 전 아래 명령어가 자동 실행될 수 있다.

```bash
pnpm lint && pnpm typecheck && pnpm build
```

- 셋 중 하나라도 실패하면 푸시가 차단될 수 있다.
